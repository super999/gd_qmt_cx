# 文档使用说明

## 目的

本文件用于告诉后续 agent / 新会话：

- 哪些文档是当前项目的权威事实来源
- 哪些文档是动态计划
- 哪些文档只是历史草稿、旧判断或可参考材料

这样可以减少两个常见问题：

1. 把过时计划当成当前任务
2. 把历史分析稿当成当前环境事实

## 建议阅读顺序

如果是新会话第一次进入本项目，建议按下面顺序读取：

1. `AGENTS.md`
2. `报告/README.md`
3. `报告/项目管理/文档使用说明.md`
4. `报告/环境与规范/MiniQMT_API可用性清单.md`
5. `报告/环境与规范/回测策略开发约束.md`
6. `报告/环境与规范/回测数据口径与指标处理约束.md`
7. `报告/项目管理/当前任务与阶段计划.md`

如果任务涉及示例策略源码分析，再补读：

8. `python/多因子选股回测示例.py`

## 文档权威级别

### A 级：当前权威事实文档

这些文档优先级最高，后续实现、分析、判断应优先以它们为准。

#### 1. `AGENTS.md`

用途：

- 项目长期说明书
- 稳定环境事实
- 工作规则
- 持久参考文档入口

特点：

- 不应频繁写入阶段性计划
- 应保持稳定、克制、长期有效

#### 2. `报告/环境与规范/MiniQMT_API可用性清单.md`

用途：

- 当前环境下 `xtquant` / `xtdata` API 的实测真相表
- 记录哪些接口可用、不可用、阻塞、未测试
- 记录关键返回值格式

特点：

- 只记录已验证事实
- 发生新测试结果时优先更新它

#### 3. `报告/环境与规范/回测策略开发约束.md`

用途：

- 告诉后续 agent 当前回测开发边界是什么
- 告诉后续 agent 第一版应该优先怎么做

#### 4. `报告/环境与规范/回测数据口径与指标处理约束.md`

用途：

- 固化复权、指标、财务数据处理口径
- 防止新会话重复讨论这些基础问题

### B 级：当前动态计划文档

这些文档反映“当前阶段想做什么”，但它们会变化。

#### `报告/项目管理/当前任务与阶段计划.md`

用途：

- 记录当前主线任务
- 记录最近阶段的优先顺序
- 记录暂缓事项

特点：

- 可随项目推进而频繁修改
- 如果计划变化，应优先改它

### C 级：当前工作输入文件

这些文件不是“规则文档”，而是当前分析/实现对象。

#### `python/多因子选股回测示例.py`

用途：

- 当前示例策略的可读源码副本

注意：

- 它是当前分析对象，不是计划文档
- 也不自动等于 QMT 实际运行加载的最终版本
- 凡涉及“最终在 QMT 中是否生效”，仍需验证加载路径与运行结果

### D 级：历史草稿 / 历史计划 / 历史分析材料

这些文件可以参考，但默认不能直接当当前事实来源。

典型包括：

- `归档/外部AI历史材料/Gemini的计划/...`
- `归档/外部AI历史材料/ChatGPT的计划/...`
- `报告/归档/可转交的分析报告.md`
- `报告/归档/分析报告的提示词.md`

它们的定位：

- 可作为思路参考
- 可作为历史决策痕迹
- 不能默认覆盖 A 级文档中的当前结论

## 当前项目中的默认判定规则

如果多个文件出现冲突，默认按下面优先级处理：

1. 代码与最新实测结果
2. A 级文档
3. B 级文档
4. C 级输入文件的上下文说明
5. D 级历史材料

## 对后续 Agent 的明确要求

后续 agent 在开始工作前，应先判断这次任务属于哪一类：

### 1. 环境/API/回测基础设施问题

优先读：

- `AGENTS.md`
- `报告/环境与规范/MiniQMT_API可用性清单.md`
- `报告/环境与规范/回测策略开发约束.md`
- `报告/环境与规范/回测数据口径与指标处理约束.md`

### 2. 当前做什么的问题

优先读：

- `报告/项目管理/当前任务与阶段计划.md`

### 3. 示例策略理解或修改问题

优先读：

- `python/多因子选股回测示例.py`
- 再结合 A 级文档判断环境和实现边界

## 维护规则

如果项目后续新增关键文档，应把它归类到本文件里，并说明它属于：

- A 级：权威事实
- B 级：动态计划
- C 级：当前输入对象
- D 级：历史参考材料
