## 3. 重要：关于复权的说明

⚠️ **复权方式会影响回测结果！**

### 3.1 什么是复权？

股票/ETF 分红后，价格会出现跳空（除权）。例如：

- 分红前价格：4.500 元
- 分红：0.100 元/份
- 分红后价格：4.400 元（跳空 -0.100 元）

**如果使用不复权数据：**
1. K 线会出现巨大的跳空缺口
2. 均线计算会严重失真
3. 可能错误触发买卖信号

### 3.2 当前策略的复权设置

| 平台 | 复权方式 | 说明 |
|------|----------|------|
| **xtquant 独立回测** | ✅ 前复权 | 使用 `dividend_type='front'` |
| **QMT** | ⚠️ 需确认 | 默认可能使用不复权数据 |

### 3.3 如何确认 QMT 的复权设置？

在 QMT 中运行策略时，需要确认：
1. 数据是否使用前复权
2. 或者手动检查是否有价格跳空

### 3.4 本次回测的特殊情况

在 2025-04-22 至 2026-04-22 期间：
- 前复权 vs 不复权的结果**完全相同**
- 原因可能是：此期间 510300 没有分红，或分红金额很小

**但这不代表复权不重要！** 在更长的时间区间或有大额分红时，影响会非常大。

### 3.5 最佳实践

1. **始终显式指定复权方式**，不要依赖默认值
2. **xtquant 中**：使用 `dividend_type='front'` 参数
3. **QMT 中**：确认数据复权设置
4. **跨平台对比时**：确保两边使用相同的复权方式

## 4. 重要：关于技术指标函数的说明

⚠️ **QMT 和 xtquant 都没有内置的 `MA()`, `EMA()`, `MACD()` 等技术指标函数！**

### 4.1 常见的误解

很多用户以为：
- QMT 中有 `MA()`, `EMA()`, `MACD()`, `KDJ()` 等函数
- xtquant 中也有类似的函数

**实际情况：**
- 两边都没有这些内置函数
- 都需要手动获取历史数据，然后用 `numpy` 或 `pandas` 计算

### 4.2 我们的实现方式（两边通用）

#### 均线计算

**QMT 版本：**
```python
# 获取历史收盘价
close_data = ContextInfo.get_history_data(61, '1d', 'close', 3)
close_list = close_data[stock]

# 手动计算 MA20 和 MA60
ma20 = np.mean(close_list[-21:-1])  # 昨天的 MA20
ma60 = np.mean(close_list[-61:-1])  # 昨天的 MA60
```

**xtquant 版本：**
```python
# 获取历史数据
df = data[stock_code]

# 手动计算 MA20 和 MA60
df['ma20'] = df['close'].rolling(20).mean()
df['ma60'] = df['close'].rolling(60).mean()
```

### 4.3 常用指标的手动实现方法

| 指标 | 实现方法 | 难度 |
|------|----------|------|
| **MA（移动平均）** | `np.mean()` 或 `rolling().mean()` | ⭐ 简单 |
| **EMA（指数平均）** | 手动计算加权 | ⭐⭐ 中等 |
| **MACD** | EMA(12) - EMA(26)，再算信号线 | ⭐⭐⭐ 较难 |
| **KDJ** | 计算 RSV，再算 K、D、J | ⭐⭐⭐ 较难 |
| **RSI** | 计算涨跌平均值 | ⭐⭐ 中等 |
| **BOLL** | MA + 2*STD | ⭐ 简单 |

### 4.4 这种方式的优点

1. **不依赖平台特有函数** - 代码更通用
2. **计算逻辑完全透明** - 容易调试和验证
3. **容易保持一致** - QMT 和 xtquant 之间容易对比
4. **不受版本差异影响** - 不担心 QMT 版本更新导致函数变化

### 4.5 确保两边一致的最佳实践

#### 1. 使用相同的计算逻辑

- 都用 `np.mean()` 计算均线
- 都取相同长度的历史数据（比如 61 根 K 线）

#### 2. 封装统一的指标函数

```python
def calculate_ma(prices, window):
    """统一的均线计算函数，两边都能用"""
    import numpy as np
    if len(prices) < window:
        return None
    return np.mean(prices[-window:])
```

#### 3. 验证数据一致性

- 确保两边使用相同的复权方式
- 确保两边的历史数据长度一致
- 确保两边的日期范围一致
