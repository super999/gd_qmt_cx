# 盘中模拟监控程序使用手册

## 路线位置

当前完整路线是：1 日线优质区间扫描、2 统计共同信号、3 候选买点规则回测、4 盘中真实买入口径验证、5 卖点设计；五步研究已初版闭环，本手册对应下一阶段的盘中模拟监控程序。

## 程序入口

- `code/run_qmt/intraday_low_absorb_monitor.py`
- `code/run_qmt/intraday_monitor_gui.py`
- `code/run_qmt/start_intraday_monitor_gui.bat`

当前支持两种模式：

- `replay`：历史回放，跑指定日期到指定日期的历史行情。
- `live`：实时监控，跑当前 MiniQMT 收到的最新行情。

固定策略口径：

- 标的：`510300.SH`
- 买入预警：`盘中弱势低位-量能修复`
- 候选买入提示：信号后第 2 和第 3 根 5m K 线开盘
- 模拟买入主口径：信号后第 3 根 5m K 线开盘
- 退出提示：第 3 个交易日尾盘时间退出
- 程序只输出日志和模拟提示，不自动下单

## 图形化界面

如果你不想每次输入命令，可以用图形化界面。

启动方式一：双击或运行批处理文件。

```powershell
code\run_qmt\start_intraday_monitor_gui.bat
```

启动方式二：直接运行 Python 文件。

```powershell
& 'd:\python_envs\gd_qmt_env\python.exe' 'code\run_qmt\intraday_monitor_gui.py'
```

图形界面里的按钮含义：

- `收盘后查今天`：等同于 replay 今天，并打印事件日志和买卖明细。
- `查指定日期`：使用界面里“单日日期”的值做 replay。
- `查指定区间`：使用界面里“区间开始”和“区间结束”的值做 replay。
- `查过去3个月`：使用界面里“区间结束”的日期，自动往前推 3 个月做 replay。
- `live 查一次`：等同于 `--mode live --max-loops 1`，看一次当前状态后退出。
- `启动 live 持续盯盘`：等同于 `--mode live --poll-seconds 10`，一直运行等信号。
- `停止 live`：停止当前正在运行的 live 或 replay 子进程。
- `查看 replay 事件表`：在软件内部打开表格窗口，直接查看最近一次 replay 生成的事件日志。
- `打开 replay 事件日志`：打开历史回放事件 CSV。
- `打开 replay 买卖明细`：打开历史回放模拟持仓 CSV。
- `打开 live 事件日志`：打开 live 运行时产生的事件 CSV。
- `打开输出目录`：打开程序输出目录。

图形界面不重写策略规则，只是把下面的命令包装成按钮。因此 GUI 和命令行版本使用同一套策略逻辑。

GUI 会自动保存你上一次输入的参数，下次打开时自动回填：

- 单日日期
- 区间开始
- 区间结束
- live 检查间隔秒
- replay 是否跳过下载
- 是否打印事件日志
- 是否打印买卖明细

配置文件位置：

```text
code/run_qmt/outputs/intraday_low_absorb_monitor/gui_settings.json
```

### GUI 内查看 replay 事件表

使用顺序：

1. 先点击 `收盘后查今天`、`查指定日期` 或 `查过去3个月`，让程序生成 replay 事件日志。
2. 再点击 `查看 replay 事件表`。
3. 软件会弹出一个表格窗口，显示：
   - 时间
   - 事件
   - 价格
   - 第几根 5m
   - 持仓号
   - 状态
   - 说明

如果这次 replay 没有触发信号，表格窗口会显示“无事件”。这不是错误，只表示该区间没有满足买入预警条件。

事件表支持筛选：

- 全部
- 买入预警
- 候选买入提示
- 模拟买入
- 退出提示
- 候选买入跳过
- 模拟买入跳过

### GUI 内查看 K 线图

在 replay 事件表里，双击任意一条事件，软件会打开该事件当天的 K 线窗口。

K 线窗口支持：

- 默认显示当天 `5m` K 线。
- 点击 `显示 1m` 可以切换到当天 `1m` K 线。
- 图上会标记：
  - 买入预警
  - 候选买入提示
  - 模拟买入
  - 退出提示
- 如果本地没有当天 `1m` 数据，窗口会提示“当前本地无 1m 数据”，不会报错。

注意：

- 图表用于复盘和观察，不代表自动下单。
- 图表数据来自 MiniQMT 本地行情数据。
- 当前只支持 `510300.SH`。

### GUI 的 live 状态栏

GUI 顶部有 live 状态信息：

- 最新 5m 时间
- 最新价
- 触发状态

点击 `live 查一次` 或 `启动 live 持续盯盘` 后，这些字段会根据程序输出自动刷新。

如果需要看 live 事件列表，可以点击：

```text
查看 live 事件表
```

### 查一次和一直盯盘有什么区别

`live` 有两种常用方式。

第一种：查一次马上退出。

```powershell
& 'd:\python_envs\gd_qmt_env\python.exe' 'code\run_qmt\intraday_low_absorb_monitor.py' --mode live --max-loops 1
```

意思是：

- 程序只读取一次当前最新行情。
- 读完后立刻退出。
- 适合你临时想看“现在为止有没有触发”。
- 例如明天 10:00 你打开电脑，只想看 9:30 到 10:00 有没有信号，就用这个。
- 缺点是：它退出以后就不再监控，10:01 以后如果触发，它不会提醒你。

第二种：一直开着等信号。

```powershell
& 'd:\python_envs\gd_qmt_env\python.exe' 'code\run_qmt\intraday_low_absorb_monitor.py' --mode live --poll-seconds 10
```

意思是：

- 程序不退出。
- 每 10 秒检查一次最新 5m K 线。
- 如果后面盘中触发买入预警，它会打印并写入日志。
- 适合你盘中真正盯盘使用。
- 你不想看了，就在窗口里按 `Ctrl+C` 退出。

简单记：

- 只想问“现在有没有”：用 `--max-loops 1`
- 想让程序帮你持续等信号：用 `--poll-seconds 10`

举例：

- 10:00 查一次：`--mode live --max-loops 1`
- 10:00 到收盘都让程序盯着：`--mode live --poll-seconds 10`

注意：

- `live` 只监控 MiniQMT 当前最新交易日。
- `live` 不能指定历史日期。
- 要验证历史某一天是否触发，应使用 `replay`。
- 例如验证 `20260403`：

```powershell
& 'd:\python_envs\gd_qmt_env\python.exe' 'code\run_qmt\intraday_low_absorb_monitor.py' --mode replay --start-date 20260403 --end-date 20260403 --print-events --print-trades
```

从干净状态重新开始监控：

```powershell
& 'd:\python_envs\gd_qmt_env\python.exe' 'code\run_qmt\intraday_low_absorb_monitor.py' --mode live --reset-state
```

退出 live：

```text
Ctrl+C
```

## live 输出

- 状态文件：`code/run_qmt/outputs/intraday_low_absorb_monitor/live_state.json`
- 实时事件日志：`code/run_qmt/outputs/intraday_low_absorb_monitor/live_event_log.csv`

如果当天没有触发信号，`live_event_log.csv` 可能不会新增事件，这是正常情况。

## 事件含义

| event_type | 中文 | 含义 |
| --- | --- | --- |
| `BUY_WARNING` | 买入预警 | 日线背景和盘中低位承接条件已经满足 |
| `ENTRY_CANDIDATE` | 候选买入提示 | 信号后第 2 或第 3 根 5m K 线开盘已出现 |
| `SIM_BUY` | 模拟买入 | 按主口径记录一笔模拟持仓，不代表真实下单 |
| `EXIT_REMINDER` | 退出提示 | 持仓到第 3 个交易日尾盘，提示时间退出 |
| `SKIP_ENTRY` | 候选买入跳过 | 信号太晚，当天剩余 K 线不足 |

## 当前边界

- live 当前是外部 Python 轮询版，不是 QMT 内策略版。
- live 依赖 MiniQMT 已打开并能提供行情。
- live 不自动下单，只输出模拟监控日志。
- 当前固定单标的 `510300.SH`，暂不做多标的扫描。
- 当前卖点只有第 3 个交易日尾盘时间退出提示。
