# 盘中模拟监控程序使用手册

## 路线位置

当前完整路线是：1 日线优质区间扫描、2 统计共同信号、3 候选买点规则回测、4 盘中真实买入口径验证、5 卖点设计；五步研究已初版闭环，本手册对应下一阶段的盘中模拟监控程序。

## 程序入口

- `code/run_qmt/intraday_low_absorb_monitor.py`

当前支持两种模式：

- `replay`：历史回放，跑指定日期到指定日期的历史行情。
- `live`：实时监控，跑当前 MiniQMT 收到的最新行情。

固定策略口径：

- 标的：`510300.SH`
- 买入预警：`盘中弱势低位-量能修复`
- 候选买入提示：信号后第 2 和第 3 根 5m K 线开盘
- 模拟买入主口径：信号后第 3 根 5m K 线开盘
- 退出提示：第 3 个交易日尾盘时间退出
- 程序只输出日志和模拟提示，不自动下单

## 运行前准备

1. 打开 MiniQMT。
2. 确认 MiniQMT 已登录，并且行情连接正常。
3. 进入项目目录：

```powershell
cd d:\codex_n_workspace\gd_qmt_cx
$env:PYTHONIOENCODING='utf-8'
```

## replay 历史回放

默认近一年区间：

```powershell
& 'd:\python_envs\gd_qmt_env\python.exe' 'code\run_qmt\intraday_low_absorb_monitor.py' --mode replay
```

指定日期区间：

```powershell
& 'd:\python_envs\gd_qmt_env\python.exe' 'code\run_qmt\intraday_low_absorb_monitor.py' --mode replay --start-date 20260101 --end-date 20260424
```

已经下载过数据时加速运行：

```powershell
& 'd:\python_envs\gd_qmt_env\python.exe' 'code\run_qmt\intraday_low_absorb_monitor.py' --mode replay --skip-download
```

直接在控制台打印买卖明细：

```powershell
& 'd:\python_envs\gd_qmt_env\python.exe' 'code\run_qmt\intraday_low_absorb_monitor.py' --mode replay --print-trades
```

直接在控制台打印完整事件链：

```powershell
& 'd:\python_envs\gd_qmt_env\python.exe' 'code\run_qmt\intraday_low_absorb_monitor.py' --mode replay --print-events
```

限制控制台最多打印 20 条：

```powershell
& 'd:\python_envs\gd_qmt_env\python.exe' 'code\run_qmt\intraday_low_absorb_monitor.py' --mode replay --print-trades --print-events --print-limit 20
```

打印全部明细：

```powershell
& 'd:\python_envs\gd_qmt_env\python.exe' 'code\run_qmt\intraday_low_absorb_monitor.py' --mode replay --print-trades --print-events --print-limit 0
```

replay 用途：

- 检查指定区间里程序在哪些时间发出预警。
- 检查信号后第 2/3 根 5m K 线买入提示是否正确。
- 检查第 3 个交易日退出提示是否正确。
- 做历史复盘，不用于实时交易。

## live 实时监控

正常启动：

```powershell
& 'd:\python_envs\gd_qmt_env\python.exe' 'code\run_qmt\intraday_low_absorb_monitor.py' --mode live
```

每 10 秒检查一次：

```powershell
& 'd:\python_envs\gd_qmt_env\python.exe' 'code\run_qmt\intraday_low_absorb_monitor.py' --mode live --poll-seconds 10
```

只测试一次连接：

```powershell
& 'd:\python_envs\gd_qmt_env\python.exe' 'code\run_qmt\intraday_low_absorb_monitor.py' --mode live --max-loops 1
```

指定 live 调试日期：

```powershell
& 'd:\python_envs\gd_qmt_env\python.exe' 'code\run_qmt\intraday_low_absorb_monitor.py' --mode live --live-date 20260427 --max-loops 1
```

注意：

- `--live-date` 只能指定当前 live 5m 数据窗口里已有的日期。
- 它不能用来模拟很早以前的历史日期。
- 例如当前最新行情是 `20260427`，你指定 `--live-date 20260403`，通常看不到信号，因为 live 只拉取最近一小段 5m 数据，不会自动回放 `20260403`。
- 要验证 `20260403` 是否触发，应使用 replay：

```powershell
& 'd:\python_envs\gd_qmt_env\python.exe' 'code\run_qmt\intraday_low_absorb_monitor.py' --mode replay --start-date 20260403 --end-date 20260403 --print-events --print-trades
```

从干净状态重新开始监控：

```powershell
& 'd:\python_envs\gd_qmt_env\python.exe' 'code\run_qmt\intraday_low_absorb_monitor.py' --mode live --reset-state
```

退出 live：

```text
Ctrl+C
```

## live 输出

- 状态文件：`code/run_qmt/outputs/intraday_low_absorb_monitor/live_state.json`
- 实时事件日志：`code/run_qmt/outputs/intraday_low_absorb_monitor/live_event_log.csv`

如果当天没有触发信号，`live_event_log.csv` 可能不会新增事件，这是正常情况。

## 事件含义

| event_type | 中文 | 含义 |
| --- | --- | --- |
| `BUY_WARNING` | 买入预警 | 日线背景和盘中低位承接条件已经满足 |
| `ENTRY_CANDIDATE` | 候选买入提示 | 信号后第 2 或第 3 根 5m K 线开盘已出现 |
| `SIM_BUY` | 模拟买入 | 按主口径记录一笔模拟持仓，不代表真实下单 |
| `EXIT_REMINDER` | 退出提示 | 持仓到第 3 个交易日尾盘，提示时间退出 |
| `SKIP_ENTRY` | 候选买入跳过 | 信号太晚，当天剩余 K 线不足 |

## 当前边界

- live 当前是外部 Python 轮询版，不是 QMT 内策略版。
- live 依赖 MiniQMT 已打开并能提供行情。
- live 不自动下单，只输出模拟监控日志。
- 当前固定单标的 `510300.SH`，暂不做多标的扫描。
- 当前卖点只有第 3 个交易日尾盘时间退出提示。
