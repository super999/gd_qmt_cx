# 报告目录说明

## 目的

本目录用于保存需要长期保留、可审计、可复用的项目文档与研究结论。  
如果某个文件只是脚本临时输出、可随时重跑，不应优先放在这里。

建议先读：

1. `项目管理/文档使用说明.md`
2. `项目管理/当前任务与阶段计划.md`
3. 再根据任务进入对应子目录

## 目录分工

### `项目管理/`

放当前项目的管理类文件：

- 当前任务与阶段计划
- 文档使用说明
- 输出文件管理与版本控制指南
- 其他索引类文件

这类文件回答的是：

- 现在在做什么
- 哪些文档权威
- 哪些输出该保留

### `环境与规范/`

放长期有效的环境事实与开发约束：

- MiniQMT API 可用性清单
- 回测策略开发约束
- 回测数据口径与指标处理约束
- 特征字段中文对照

这类文件回答的是：

- 当前环境里哪些 API 能用
- 回测里复权、指标、财务数据怎么处理
- 后续报告里的特征字段是什么意思

### `策略设计/`

放当前策略路线、说明书、策略卡片：

- 主路线执行计划
- 策略说明书
- 当前实际运行策略卡片

这类文件回答的是：

- 当前策略想做什么
- 当前代码实际在干什么
- 理想设计与当前实验版本差在哪里

### `研究计划/`

放研究类任务的定义与计划，不直接存结果：

- 事件定义与研究计划
- 多时间框架研究计划

这类文件回答的是：

- 研究准备怎么做
- 标签怎么定义
- 下一轮分析会看什么

### `研究结论/当前主线/`

放当前主研究方向的结论性 Markdown 报告。  
这些文件应优先保留，并可以提交 Git。

当前这里主要是：

- `510300` 近一年日线最优交易区间扫描
- `510300` 优质区间买入日信号统计分析
- `510300` 单标研究结论
- 事件扫描结果
- 事件前特征分析
- 事件日前窗口剖面分析
- 多时间框架 V 字结构分析
- `n5_r3` 评分模型
- 背景分与触发分拆分模型
- `n5_r3` 候选规则首轮回测结果
- `n5_r3` 盘中即时信号回放结果
- `n5_r3` 盘中即时信号逐笔复盘
- `n5_r3` 盘中即时信号简单复盘

### `研究结论/数据摘要/`

放结论报告对应的轻量摘要 CSV。  
这些文件用于：

- 让后续 agent 快速读摘要
- 做人工复核
- 保留关键表格而不保存全部原始中间数据

### `研究结论/历史探索/`

放已经完成、但不再属于当前主线的历史性探索结论：

- ETF 横向对比
- 参数稳健性扫描

这类文件可以参考，但不应默认当作当前主路线结论。

### `源码分析/`

放对示例策略源码的分析。

### `归档/`

放历史材料、旧版分析稿、可转交材料。  
默认不是当前权威事实来源。

## 什么应该放进 Git

建议提交 Git：

- 策略文档
- 研究计划
- 研究结论 Markdown
- 研究摘要 CSV
- 环境与规范类文档

通常不必提交 Git：

- `code/backtest/outputs/` 下的原始运行输出
- 大体量中间数据集
- 可从脚本重跑得到的临时产物

## 如何判断一个文件是不是“当前有效”

优先级按下面看：

1. `项目管理/当前任务与阶段计划.md`
2. `策略设计/当前实际运行策略卡片.md`
3. `研究结论/当前主线/`
4. `研究结论/历史探索/`
5. `归档/`

如果一个文件在 `归档/` 或 `研究结论/历史探索/`，默认不要把它当当前主结论。
