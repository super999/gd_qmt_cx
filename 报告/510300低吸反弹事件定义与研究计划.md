# 510300 低吸反弹事件定义与研究计划

## 文档定位

本文件用于定义：

1. `510300.SH` 的“低吸反弹事件”如何打标签
2. 第一阶段事件研究要产出什么数据
3. 后续数学分析和建模按什么顺序推进

这份文档不是策略说明书。  
它的目标不是直接告诉我们“该怎么买”，而是先把历史上哪些窗口可以被视为“有效低吸反弹”定义清楚，然后再从样本中找规律。

## 当前研究对象

- 标的：`510300.SH`
- 时间范围：`2024-01-01` 到 `2026-04-24`
- 数据口径：前复权日线数据

说明：

- 当前阶段先用日线中的 `low / high / close` 来定义事件标签
- “盘中低点”在这一阶段先用日线 `low` 近似承接
- 后续若要更细化，再把 `30m / 5m` 结构并入特征

## 核心研究思路

当前研究顺序固定为：

1. 先定义“有效低吸反弹窗口”
2. 再把 `2024-2026` 中所有候选事件扫出来
3. 再分析这些事件的共同规律
4. 最后才把这些规律翻译成策略条件和参数

也就是说：

- 当前不是先拍一个策略再调参数
- 而是先找样本，再建模，再反推策略

## 事件标签定义 v1

### 1. 候选低点日

对任意交易日 `t`，如果满足：

- `low[t]` 是过去 `K` 个交易日中的最低点之一

则把 `t` 视为一个候选低点日。

当前基线参数：

- `K = 5`

说明：

- 这里只要求它在过去短窗口里具有局部低点意义
- 不要求它一定是更大级别周期的绝对低点

### 2. 有效反弹窗口

对候选低点日 `t`，记当日低点为：

- `L_t = low[t]`

再定义一个未来观察窗口：

- 从 `t` 日开始，连续观察未来 `N` 个交易日
- 这里的 `N` **包含事件日 `t` 本身**

如果在这 `N` 天内同时满足：

1. 最低价始终不低于 `L_t * (1 - eps)`
2. 最高价至少达到 `L_t * (1 + R)`

则把 `t` 标记为一次“有效低吸反弹事件”。

### 3. 参数含义

- `N`：事件成立的观察窗口长度
- `R`：从事件低点出发的最低目标反弹幅度
- `eps`：允许的低点容差

当前基线参数：

- `eps = 0.001`，即允许未来最低价最多比事件低点再低 `0.1%`

说明：

- 加容差是为了避免因为极小价差把原本合理的事件错误打成失败

## 第一轮并行标签方案

第一轮不只做一种标签，先并行打 4 组：

1. `N = 5, R = 3%`
2. `N = 5, R = 5%`
3. `N = 10, R = 3%`
4. `N = 10, R = 5%`

这样做的目的：

- 看不同标签下事件数量差多少
- 看哪一种标签更接近你想做的“短线反弹”
- 为后续建模准备多套样本定义，而不是一开始就把标签钉死

## 当前样本输出要求

第一轮扫描脚本至少要输出：

### 1. 全部候选低点日

每一行对应一个候选交易日，至少包含：

- `trade_date`
- `low`
- `high`
- `close`
- 是否是过去 `K` 日局部低点
- 每组 `(N, R)` 标签下是否成立
- 每组标签下未来窗口最低价 / 最高价 / 最大反弹

### 2. 正样本事件表

只保留被标记为“有效低吸反弹”的样本，至少包含：

- 事件日
- 事件低点
- 观察窗口参数
- 是否守住低点
- 是否达到目标反弹
- 首次达到目标的日期
- 未来窗口最高反弹幅度

### 3. 去重后的事件窗口

因为相邻交易日可能连续满足条件，所以需要输出一份“去重事件窗口表”。

当前去重规则先用最简单版本：

- 对同一组 `(N, R)`，如果多个正样本事件窗口重叠
- 只保留**最早出现的那个事件日**

说明：

- 这不是最终规则
- 但足够让第一轮样本分析不被大量重复样本污染

## 第一轮不做的事情

当前阶段明确不做：

1. 直接从这些标签推导买卖策略
2. 直接做参数优化
3. 直接做复杂黑盒模型
4. 把盘中 `30m` 结构直接并入标签定义

## 第一轮数据分析目标

在样本扫出来之后，优先分析这些问题：

1. 各组标签下，总共有多少有效事件
2. 这些事件集中发生在哪些年份 / 哪些市场状态
3. 事件发生前的回撤深度通常多大
4. 事件发生前的下跌持续时间通常多长
5. 事件发生时相对均线的位置如何
6. 事件前后的成交量和波动率有什么特征

## 建模顺序

后续建模顺序固定为：

### 第 1 步：描述统计

- 样本数量
- 事件分布
- 关键变量分布

### 第 2 步：正负样本对比

- 有效事件 vs 无效事件
- 看哪些变量最有区分度

### 第 3 步：简单可解释模型

优先考虑：

- 分组打分
- 逻辑回归
- 简单决策树

### 第 4 步：反推策略

只有在前面 3 步完成之后，才进入：

- 条件筛选
- 阈值收敛
- 滚动校准设计

## 当前一句话结论

当前阶段不是“直接优化低吸策略”，而是：

**先把 `510300.SH` 历史上哪些窗口可以算作有效低吸反弹事件找出来，再用这些样本去分析规律、建模，最后才反推策略。**
