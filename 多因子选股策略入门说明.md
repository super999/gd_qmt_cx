- 它什么时候买
- 它什么时候卖
- 它现在有哪些问题
- 你以后最适合先改什么

对应的策略文件在这里：

## 1. 先说结论：这个策略到底是什么

这是一个“定期选股票、定期换仓”的回测策略。

你可以把它理解成这样一套流程：

1. 先在沪深300成分股里选股票
2. 每隔一段时间看一次，哪些股票值得买，哪些股票该卖
3. 对“值得买”的股票再做一次排序
4. 最多选前 10 只
5. 每只股票大概分 10% 的资金去买
6. 对不符合条件的旧持仓卖掉

所以，它不是“盯着一只股票不断高频交易”的策略。
它更像是“每隔一段时间，重新挑一批股票来持有”的策略。

---

## 2. 这个策略用的是哪些股票

代码里有这一句：

```python
ContextInfo.s = ContextInfo.get_sector('000300.SH')
```

意思是：

- 取沪深300成分股
- 把这些股票当作策略的观察范围

简单理解：

- 这个策略不是在全市场乱选
- 它只在沪深300这一批股票里选

这样做的好处是：

- 股票范围更明确
- 适合做教学示例
- 回测时不至于太散

---

## 3. 它多久调一次仓

代码里是：

```python
if d > 60 and d % 20 == 0:
```

你可以先简单理解成：

- 前 60 根日线先不动，先留出足够历史数据
- 之后大约每 20 个交易日调一次仓

因为这份策略跑的是日线，所以：

- 20 个交易日，差不多就是 1 个月左右

所以它不是每天都换仓，而是“按月调仓”的感觉。

---

## 4. 它是怎么判断“可以买”的

买入条件写在 `signal(ContextInfo)` 里面。

关键逻辑是：

```python
if data_high_pre[k][-2] > max(data_high[k][:-2]):
    buy[k] = 1
```

简单翻译成人话就是：

- 如果这只股票前一段时间的高点，突破了过去 20 天的高点
- 那就把它放进“可以买入的备选名单”

你可以把它理解成：

- 这只股票最近比较强
- 强到创出了一个阶段新高
- 所以先把它列入候选

注意：

- 这里只是“进入买入备选”
- 不是说进了备选就一定会买

---

## 5. 它是怎么判断“该卖”的

卖出条件也是在 `signal(ContextInfo)` 里面。

关键逻辑是：

```python
elif data_high_pre[k][-2] < np.mean(data_close60[k][:-2]):
    sell[k] = 1
```

简单理解：

- 如果这只股票表现变弱
- 弱到低于过去 60 天的平均收盘水平
- 那就把它放进“卖出备选名单”

你可以把它理解成：

- 股票走弱了
- 不像之前那么强
- 所以准备考虑卖出

---

## 6. 为什么它叫“多因子选股”

因为它不是只看“突破”这一个条件。

代码里在买入备选股票上，又做了一次因子排名：

```python
rank1[k] = ext_data_rank('atr',k[-2:]+k[0:6],0,ContextInfo)
rank2[k] = ext_data_rank('adtm',k[-2:]+k[0:6],0,ContextInfo)
```

这里至少出现了两个因子：

- `atr`
- `adtm`

你现在不用先深究它们的数学定义。
先把它理解成：

- 这是两个“评价股票好坏”的分数来源
- 策略本来想用这两个分数来帮忙筛选股票

但是要注意一个很重要的问题：

### 这份代码虽然算了两个因子，但实际上只用了一个

因为后面写的是：

```python
rank_total[k] = 1.0 * rank1[k]
```

这说明：

- `rank2` 算出来了
- 但没有真正参与最终总分

所以这份代码现在更准确地说：

- 名字叫多因子
- 但实际排序时更像“单因子为主”

这是后面非常值得改的地方。

---

## 7. 它最后买哪些股票

策略先把可以买的股票挑出来，再按分数排序：

```python
tmp = sorted(list(rank_total.items()), key = lambda item:item[1])
```

然后：

```python
if len(tmp) >= 10:
    tmp_stock = {i[0] for i in tmp[:10]}
```

这表示：

- 如果候选很多，就只拿前 10 只
- 如果候选不够 10 只，就有多少买多少

所以最终买入池的意思就是：

- 先满足买入条件
- 再按因子分数筛一次
- 最后只保留前 10 只

---

## 8. 它是怎么分配资金的

代码里有：

```python
ContextInfo.weight = [0.1] * 10
```

这表示：

- 最多按 10 只股票来分配
- 每只股票分配 10%

后面又写了：

```python
ContextInfo.money_distribution = {k:i*ContextInfo.money for (k,i) in zip(tmp_stock,ContextInfo.weight)}
```

简单理解：

- 当前账户还剩多少钱
- 每只目标股票分 10%
- 然后计算每只股票大约可以买多少股

后面的下单股数是：

```python
order[k] = int(ContextInfo.money_distribution[k]/(price[k][-1]))/100
order_shares(k,order[k]*100,'fix',price[k][-1],ContextInfo,ContextInfo.accountID)
```

它的意思是：

- 先用“分到的钱 ÷ 当前价格”算出大概可以买多少股
- 再按 100 股一手来处理
- 最后下单

所以从代码上看：

### 这个策略的买入逻辑是完整的

它不是只有：

```python
print('ready to buy')
```

它后面确实有真实的：

```python
order_shares(...)
```

这也和你回测界面右侧看到的真实买卖记录是一致的。

## 9. 它什么时候真正卖出

卖出代码在这里：

```python
if ContextInfo.holdings[k] > 0 and sells[k] == 1:
    order_shares(k,-ContextInfo.holdings[k]*100,'fix',price[k][-1],ContextInfo,ContextInfo.accountID)
```

意思是：

- 如果这只股票我现在有持仓
- 并且它进入了卖出备选
- 那就把这只股票全部卖掉

所以这个策略卖出不是“卖一点点”，而是：

- 满足卖出条件就清仓

## 10. 为什么你之前觉得“它可能没买”

因为它的日志写得太简单了。

现在的日志大概只有：

- 调仓日期
- `ready to buy`
- 股票代码
- `ready to sell`
- 股票代码
- 最后一行资金数字

这会导致你看日志的时候容易有这些感觉：

- 它只是说“准备买”
- 但我看不出来到底买了多少
- 我也看不出来是在哪一天正式成交
- 更看不出来这次调仓前后持仓发生了什么变化

但从源码和你之前的回测截图一起看，结论是：

### 它确实有真实买卖，不是只打印不下单

问题不在“有没有下单”，而在：

### 日志不够清楚，不适合新手学习

## 11. 这份策略现在最明显的几个问题

这里我尽量用最直白的话说。

### 问题 1：名义上是多因子，实际上只用了一个主因子

代码算了：

- `atr`
- `adtm`

但最后只用了：

```python
rank_total[k] = 1.0 * rank1[k]
```

所以现在并不是真正的“双因子加权”。

### 问题 2：日志太少

它没有清楚打印：

- 本次调仓日期
- 本次候选池
- 本次最终买入池
- 哪些股票被卖出
- 每只股票买了多少股
- 每只股票卖了多少股
- 调仓前后资金变化

这会让你学起来很痛苦。

### 问题 3：资金计算值得后面重点复核

买入时它写的是：

```python
ContextInfo.money -= price[k][-1] * order[k] * 100 - 0.0003*order[k]*100*price[k][-1]
```

这行从“账户资金减少”的直觉来看，后面很值得再检查。

你现在先不用自己改。
先记住：

- 这里是以后要重点核对的地方

### 问题 4：有些变量写了但没充分发挥作用

比如：

- `rank2` 算了没参与总分
- `ContextInfo.day` 初始化了，但基本没用

这说明这份示例是“能跑”，但还不算“写得很完整很清晰”。

## 12. 如果你运行这个策略，它到底做了什么

你以后看回测结果，可以按下面这个顺序去理解，不用一上来就看一堆专业指标。

### 第一步：看它是不是到调仓日了

因为它不是每天都动。
它大约每 20 个交易日才动一次。

### 第二步：看哪些股票进入买入备选

进入买入备选的意思是：

- 这些股票最近比较强
- 先进入候选名单

### 第三步：看哪些股票进入卖出备选

进入卖出备选的意思是：

- 这些股票变弱了
- 可能要被清仓

### 第四步：看候选股票的排序结果

不是所有候选都会买。
它会按因子分数再排一次序。

### 第五步：看最终买入前 10 名

最后只买前 10 只左右。

### 第六步：看调仓后持仓变成什么样

真正重要的是：

- 原来持有哪些股票
- 现在卖掉了哪些
- 新买了哪些
- 账户还剩多少现金

如果这些东西在日志里看不出来，说明这份策略还不够适合学习。

## 13. 你以后最适合先改哪些内容

如果你的目标是最后写出自己的 QMT 策略，我建议你按下面顺序改，不要一开始就搞得很复杂。

### 第一优先：先把日志改清楚

因为如果日志不清楚：

- 你根本不知道策略为什么买
- 你也不知道它为什么卖
- 更没法判断自己改对没改对

### 第二优先：把因子总分逻辑改清楚

现在最明显的问题是：

- 名义上两个因子
- 实际只用了一个

后面最适合先改成：

- 真正的双因子加权
- 或者先明确只保留一个因子

### 第三优先：核对资金和收益的计算

这一步非常重要，因为：

- 买卖逻辑可以先简单
- 但资金算错会导致回测结果失真

### 第四优先：再改成你自己的买卖规则

等你把原策略看懂后，再去改：

- 股票池
- 因子
- 调仓周期
- 买卖条件
- 风控规则

这样不容易把自己绕晕。

## 14. 对你这个新手来说，最重要的理解方式

你现在不用先去背很多量化术语。

你只要把这个策略先理解成一句话：

> 它每隔一段时间，在沪深300里找一批最近比较强的股票，再从里面按因子排个序，只买前 10 个，同时把已经变弱的旧持仓卖掉。

如果你能先把这句话理解稳了，后面再去看代码就会轻松很多。
